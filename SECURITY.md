# Security Policy

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Send a private report with reproduction details and impact summary to the
maintainers through repository security advisories (preferred), or contact the
maintainer directly if advisories are not enabled.

Include:

- affected version
- environment details
- reproduction steps or proof of concept
- impact and suggested mitigation (if known)

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

Older versions may not receive security fixes.
